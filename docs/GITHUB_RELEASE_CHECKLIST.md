# GitHub Release Checklist

## Repository Readiness
- README with setup and usage
- LICENSE
- CONTRIBUTING guidelines
- CI workflow running tests and benchmark script
- Deployment guide
- API specification
- Architecture documentation
- Role-based user guide

## Before Publishing
1. Remove local-only artifacts and secrets
2. Verify .env is ignored and .env.example is accurate
3. Run tests and benchmark locally
4. Push to GitHub and verify CI status
5. Create release notes with feature summary and risk notes

## Delivery Notes
- Demo video is prepared via docs/DEMO_VIDEO_SCRIPT.md instructions.
- Actual recorded video file must be captured manually.
